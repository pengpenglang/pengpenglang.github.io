数列分块讲解

数列的操作有很多种算法（例如线段树、树状数组、ST表等），他们的时间复杂度都是优秀的的 log 级别，但是这些高级的数据结构都避不开一个问题——内存空间的高额占用。其次这些数据结构的实现都比较复杂难写，因此我们在优化暴力的基础上另一思想——数列分块处理。

分块算法可以说是一种偏数据结构类的通用算法，没有艰深的内容，与暴力的思想最为相似，但是在很多的题目中都可以发挥作用。

相关概念

  • 块：我们将数列划分成若干个不相交的区间，每个区间称为一个块
  • 整块：在一个区间操作时，完整包含于区间的块
  • 不完整的块：在一个区间操作时，只有部分包含于区间的块，即区间左右端点所在的两个块

使用范围

分块算法是折半查找和顺序查找的一种改进算法，分块算法由于只需要索引表是有序的，对块内节点没有排序的要求，因此特别适合节点的动态变化的情况

主要思想

整块间接维护标记，局部直接朴素修改

为什么局部不能也先给对应的块标上信息最后一次性修改呢？因为这样还需要记录标上信息的块哪些元素需要更新，哪些元素不需要更新，这样做反而南辕北辙了

一般将数列划分为几个块进行处理，每个块有一个标记用来记录整个块内元素都需要进行的操作。在进行操作的时候如果整个块都包含那么只修改标记不修改（类似线段树的懒标记），不完整块内的元素直接暴力修改。最后查询某个元素的时候只需要将标记信息处理就是结果。如此整块内的元素修改操作明显减少起到了优化的作用，并且减少了无用修改的操作（只有查询的时候将标记计算）

数列分块例题

这里以hzwer的1~9数列分块入门来进一步具体讲解思路用法，另外还参考了这个博客的前四道题的代码和另一篇博客

题 1

数列分块入门 1：给出一个长为 n 的数列，以及 n 个操作，操作涉及区间加法，单点查值

无疑这题也可以用树状数组和线段树来解决，下面讲讲分块的基本思路。分块就是将原本数列的 n 个数分为sqrt(n)个块，对于区间操作我们可以看成是对每一个涉及的块进行分别操作。

16 分为 4 个每个包含 4 个元素的块。26 分为 5 个每个包含 5 个元素的块，剩余的在自己组成一个块合计 6 个块，即分块是向上取整的

具体做法如下：

  • 预处理：将 n 个元素分为 t=sqrt(n)个块，其中第 i 个块的覆盖范围为 [(i−1)∗t+1,i∗t] ，对于剩下的不完整的块额外组成一个块范围为 [n−[n/t]∗t+1,n] ，对于元素开一个数组 pos[i] 记录第 i 个元素所处的块的编号，对于块开 L[] 和 R[] 记录每个块的端点元素的下标，再根据题意决定如何存储每个块的更新信息，这里开一个 changed[] 来记录整块的加法信息
  • 修改操作：对于每次加法操作区间包含的部分一定是若干个完整的块（可能是 0 ）和之多两个不完整块，对于不完整块暴力扫描直接修改元素，对于完整的块先将加法的信息存入 changed[]
  • 查询操作：查询某个元素的值是只用再将其所处的块的加法信息加上就是最终结果

总之要深深的领悟分块的真谛：整块维护，局部朴素

题 2

数列分块入门 2：给出一个长为 n 的数列，以及 n 个操作，操作涉及区间加法，询问区间内小于某个值 x 的元素个数

只用在上述的基础上另开一个 vector 记录每个块内的元素的有序序列

  • 预处理：在上述基础上将元素出入对应的 vector 内排好序

  • 修改操作：整块加法后顺序不变，所以直接记录加法信息即可。不完整块加法后顺序改变，所以需要修改元素后重新将块排序

  • 查询操作：完整块用二分查找，不完整块只能暴力枚举统计

题 3

数列分块入门 3：给出一个长为 n 的数列，以及 n 个操作，操作涉及区间加法，询问区间内小于某个值 n 的前驱（比其小的最大元素）

题二变式，还是需要用 vector 维护每一个块内元素的顺序，但是这次找的是指定值的前驱

  • 预处理和修改操作完全和题 2 操作一样

  • 查询操作：整块的部分用二分查找找前驱尝试更新答案，不完整块暴力扫描找尝试更新答案

题 4

数列分块 4：给出一个长为 n 的数列，以及 n 个操作，操作涉及区间加法，区间求和

分块的一种典型问题，为了快速查询区间和我们另开一个 sum[] 来记录每个块的元素和

  • 预处理：题一的基础上还要预处理获得每个块的 sum
  • 修改操作：完整块记录加法信息，不完整块直接修改元素的值和所处块的sum
  • 查询操作：区间内完整块的值就是 sum+加法信息 ，不完整块直接将元素值和所处块的加法信息加和到答案

题 5

数列分块入门 5：给出一个长为 n 的数列 a1…an ，以及 n 个操作，操作涉及区间开方（向下取整），区间求和

由于开放每个元素的变化都不一致，因此不能向上述几道题一样维护将整块加法信息，不过最大数据是 231 ，最多开放方 5 次值就变为 0/1 以后再修改也不会再变化，所以优化是：如果每次区间开方只不涉及完整的块，意味着不超过 2√n 个元素，直接暴力即可。如果涉及了一些完整的块，这些块经过几次操作以后就会都变成 0/1 ，于是我们采取一种分块优化的暴力做法只要每个整块暴力开方后，记录一下元素是否都变成了 0/1 ，区间修改时跳过那些全为 0/1 的块即可

  • 预处理：在题四的基础上另开一个 vis[] 标记块内部元素是否都已经变为 0/1

  • 修改操作：对于不完整块只需要修改元素，完整块修改完后判断当前块内部元素是否全部变为 0/1 ，是的话就标记 vis[] 为 1 下次再遇见当前块操作直接跳过

  • 查询操作：和题 4 完全一致

题 6

数列分块入门 6：给出一个长为 n 的数列，以及 n 个操作，操作涉及单点插入，单点询问，数据随机生成

动态插入问题，对于随机数据插入我们可以用 vector 分别记录每一个块的元素，如此先借助查询找到合适的位置然后暴力插入即可，块内其他元素位置向后移一位。但是有一个问题就是若是数据恰好很特殊，大量的点插入到同一个块内，那么块的大小会大大超过 √n ，再在块内暴力插入复杂度会变得非常糟糕失去了分块优化的作用。因此再引入另一个操作：重新分块（重构），每√n次插入后就将块的元素合并重新分块，重构需要的复杂度为 O(n) ，重构的次数为√n，所以重构的复杂度没有问题，而且保证了每个块的大小相对均衡避免了上述特殊情况。

  • 预处理：分块并将每个块的元素存到 vector ，另外还要声明 m 来实时记录数列大小，以及将 t 声明为全局变量记录最近一次分块的数量共重新分块使用
  • 修改操作：先借助查询操作找到元素在对应的 vector 的对应位置，然后用 STL 的 insert() 即可完成插入
  • 查询操作：因为要找到相应的 vector 和所处的位置，所以用 pair 返还一对数据

题 7

数列分块入门 7：给出一个长为 n 的数列，以及 n 个操作，操作涉及区间乘法，区间加法，单点询问

如果只是区间乘法，显然和题 1 一样，但是当涉及乘法是就得考虑如何同时维护加法、乘法两个标记。线段树维护两个标记各自累计就可以了因为每次到某个节点都会先下传标记然后在更新标记，但是分块处理是最终一次性处理标记所以需要考虑两者的优先级。

我们让乘法标记的优先级高于加法（如果反过来的话，新的加法标记无法处理）若当前的一个块乘以 m1 后加上 a1 ，这时进行一个乘 m2 的操作，则原来的标记变成 m1*m2,a1*m2 ，若当前的一个块乘以 m1 后加上 a1 ，这时进行一个加 a2 的操作，则原来的标记变成 m1,a1+a2 。也就是说当我们一个完整块进行加法时可以直接加到加法标记中，而当我们对一个完整块进行乘法时，需要同时对乘法标记和加法标记做乘法。当我们对于不完整块更新时，一定要先下推清除整个块的加法标记和乘法标记才能进行暴力更新

  • 预处理：开两个标记数组记录加法和乘法，注意加法数组初始化为 1
  • 修改操作：整块修改标记（注意优先级），不完整块要先下推处理整个块的标记然后在暴力更新元素
  • 查询操作：将剩余的标记信息处理即可

题 8

数列分块入门 8：给出一个长为 n 的数列，以及 n 个操作，操作涉及区间询问等于一个数 c 的元素，并将这个区间的所有元素改为 c

此题和题 5 数列开方很像，我们优化的点无疑是如果某个块内的元素的值完全一致，那么我们只更新信息，等万不得已的时候在一个一个赋值来减少修改次数。我们需要弄个标记，标记每个块内的元素是否全部为一个值，如果块内元素不是同一个值我们就暴力就行查找和修改，如果是同一个值，只需要修改标记就好了，全部为同一值的块我们可以在 O(1) 的时间内便可以统计出结果。需要注意的是，当进行部分修改时，需要先下推清除标记，以免丢失之前的标记

  • 预处理：开一个 vis[] 标记块内元素是否一致，用 tag[] 来记录块内元素相同的值
  • 修改和查询处理：两者同时进行，若是不完整的块就先下推标记信息，在修改块内元素。若是整块，先看看vis标记是否为1，若是就是 O(1) 更新 tag 即可，若不是只能暴力修改并将修改后的块标记为 vis=1,tag=c

思路应该是没问题，测了好多的数据但是交题还是 WA ，应该是某些小的细节没处理好
