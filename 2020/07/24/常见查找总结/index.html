常见查找总结
发表于2020-07-24|更新于2020-08-28|大学计算机学习C++

查找：根据给定的某个值，在查找表中确定一个关键字的等于给定值的数据元素（或记录）

查找分类

  • 静态查找

  • 动态查找：在查找表中存在删除和插入的操作

顺序查找

操作：从查找表尾开始依次将扫描到的节点关键字与给定值 value 比较，相等则结束扫描并返还下标值。若返还值为非 0 则查找成功，否则表示查找失败。

插值查找

注意：下述所有的查找方法只适用于有序顺序表

折半查找

操作：用给定的 value 先与中间节点的关键字比较，中间节点把线性表分为两个子表，若是相等则查找成功；若是不相等，再根据k与该中间节点关键字的比较结果确定下一步查找哪个子表，这样递归进行，直到查找到或者查找结果发现没有这样的节点。

写法一：非递归式

写法二：递归式（关键字，左右边界）

插值查找

引入：基于二分查找，但是做了一些改进。因为二分查找总是从中间开始找，所以每次查找都重复扫描了许多区段。

操作：可以通过自适应大致先确定关键字在有序表中所处的位置，然后在通过mid值变化靠近关键字从而间接减少了比较次数。

二分查找 mid 初始值的确定：

mid=(low+high)/2​ 即​ mid=low+(high-low)/2​

所以自适应后mid初始值的确定：mid=low+(key-a[low])/(a[high]-a[low])*(high-low)​

按数值比例确定 mid 所在的大致区段

斐波那契查找

斐波那契数列，又称黄金分割数列，指的是这样一个数列：1、1、2、3、5、8、13、21、····，在数学上，斐波那契被递归方法如下定义：F(1)=1,F(2)=1,F(n)=f(n-1)+F(n-2)(n>=2) 。该数列越往后相邻的两个数的比值越趋向于黄金比例值（ 0.618 ）

斐波那契查找就是在二分查找的基础上根据斐波那契数列进行分割的。在斐波那契数列找一个等于略大于查找表中元素个数的数 F[n] ，将原查找表扩展为长度为 F[n]（如果要补充元素，则补充重复最后一个元素，直到满足 F[n] 个元素），完成后进行斐波那契分割，即 F[n] 个元素分割为前半部分 F[n-1] 个元素，后半部分 F[n-2] 个元素，找出要查找的元素在那一部分并递归直到找到

斐波那契查找的时间复杂度还是 O(log2n) ，但是 与折半查找相比，斐波那契查找的优点是它只涉及加法和减法运算，而不用除法，而除法比加减法要占用更多的时间，因此，斐波那契查找的运行时间理论上比折半查找小，但是还是得视具体情况而定。

操作：前提是数列的长度满足某个斐波那契数，然后根据斐波那契额序列的特点对有序表进行分割

库函数查找

STL模板库中关于查找函数的：find() 、find_if() 、binary_search()

find()

作用：在区间查找序列中存在的某个关键值

格式：find(start,end,key)

函数返回值：地址或迭代器，若是查找不到key就返还end

传入参数：数据集合的左端点，数据集合的右端点后一个位置，查找的值

因为函数的查找是在 [l,r) 的左开半闭区间，所以区间要往后多开一位

find_if()

作用：在序列中找到符合某条件的第一个关键值

格式：find_if(start,end,pred)，其中 pred 是自定义的一元 bool 判断函数

函数返回值：地址或迭代器，若是查找不到 key 就返还 end

传入参数：数据集合的左端点，数据集合的右端点后一个位置，查找的值

lower_bound() 、upper_bound() 、binary_search()

注意：和上述的函数不同，这些函数使用前提是序列有序

格式：find(start,end,key)

lower_bound() upper_bound() ``binary_search()`
返还第一个大于等于 key 值的地址或迭代器 返还第一个大于 key 值的地址或迭代器 判断序列中是否存在key 值
int型 int型 bool型

配合使用可以实现二分查找,另外函数最后还可以加上比较函数使原有用法变反
