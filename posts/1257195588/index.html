C++程序设计——STL容器与迭代器
发表于2020-11-10|更新于2020-11-22|大学计算机学习C++程序设计课知识总结
|字数总计:7.4k|阅读时长:28分钟|阅读量:

STL 是 C++ 的灵魂所在，借助其函数和容器可以轻松实现许多功能，这里只能做简单的讲解，真正想要条理清楚的了解需要啃透书本

容器概览

容器的概念

  1. 标准模板库中的容器是用来存储数据集合的通用数据结构
    • 通过使用容器，用户不需要再使用 C 风格的数组、链表甚至是堆栈
    • 容器通过模板实现，可以初始化存储任意满足基本需求的任意数据类型
  2. 典型的容器

    • 顺序容器：vector list deque array forward_list

    • 关联容器：map multimap set multiset

    • 容器适配器：queue priority_queue stack

    • 无序关联容器：unordered_map unordered_set unordered_multimap unordered_multiset

    • 其他：bitset string

一般容器所在的头文件就是自己的名字

容器对元素的要求

  1. 存储值

    • 容器存储数据时，保存数据的拷贝
    • 查询某个元素是，返回对该元素的拷贝

  2. 使用容器存储时对数据类型的基本要求

    • 缺省构造函数：创建元素时调用

    • 拷贝构造函数：插入元素时调用

    • 移动拷贝构造函数：构造元素并销毁源元素时调用

    • 析构函数：清除元素是调用

    • 赋值运算符：修改某个元素时调用

    • 移动赋值运算符：赋值并销毁源元素是调用

    • operator==：比较元素时调用

    • operator<：比较元素时调用

重载这两个比较运算符就可以实现其他的比较函数

迭代器

  1. 通过迭代器提供访问容器元素的抽象方法

    • 每种容器都有基于特定容器的迭代器，可以看做是为遍历容器元素的智能指针

    实际上就是模仿指针实现的一种用于访问访问容器的媒介，本身不能输出存储的是和元素在容器中位置的相关信息，* 后可以输出其对应的元素内容

    • 不同类型迭代器往往会提供一组公共接口，提供遍历容器的“通用方法”，而容器的功能却是千差万别的
    • 类似普通数组中的指针，迭代器通过 ++ 运算移动并引用下一个元素，通过 * 或者 -> 运算符访问当前元素

  2. 迭代器类型

    • iterator：可用于读写的迭代器类型
    • const_iterator：只读访问元素的迭代器类型

  3. 与迭代器相关的容器方法

    • begin()：返还引用第一个元素的迭代器
    • end()：返还引用最后一个元素下一个位置的迭代器
    • 若是 begin() 返还值为 start，end() 返还值为 end，则容器内部的元素可用 [start,end) 的半开区间表示
    • cbegin() 和 cend() 方法返还对应的常量迭代器，只能用于访问不能修改
    • rbegin()：返还引用第一个元素前一个位置的迭代器
    • rend()：返还引用最后一个元素的迭代器

C++ 11的性能提升

  • 移动支持
  • vector 等的 push_back() 支持移动语义

顺序容器

向量 vector

特点

  • vector 容器与传统数组类似，元素在内存中连续存储
  • 每个元素都有索引，既可以用迭代器遍历也可以用下标遍历
  • 类型声明 template <typename T, typename Allocator = allocator<T> >class vector;

示例

  • 定义存放 double 的 vector，设置大小为 10

  • 通过 [] 访问容器内部的元素，无安全检查（可以越界访问）

  • 容器内部所有元素在内存中连续存储

构造与赋值

可变长度的 vector

  • 调用容器的 push_back() 向容器尾部追加元素
  • vector 的 size() 方法返还容器内部当前元素的数量

迭代器

  • 容器的 begin() 方法，返还引用第一个元素的迭代器
  • 容器的 end() 方法，返还最后一个元素的下一个位置
  • 通过 * 运算作用于迭代器，得到其所引用的元素
  • ++it 运行效率更高，返还引用而不是临时拷贝

遍历 for 循环

  • auto 遍历容器元素获得元素的拷贝，只读元素
  • auto & 遍历容器元素获得元素的引用，可以修改

迭代器的区间访问

end() 方法返还最后一个有效元素的下一个位置，进制解析并访问它，只能用于位置判断

访问 vector 元素的其他方法

  • at(int index)：访问指定下标的元素
  • front()：访问第一个元素
  • back()：访问最后一个元素

[] 运算符不进行下标越界访问，但是 at() 方法检查下标，若是越界话会抛出 out_of_range 异常

插入与删除元素

插入元素

向任意位置插入元素都是借助 insert()

  • 版本一：一次插入一个元素
  • 版本二：一次插入指定值的多个拷贝
  • 版本三：插入迭代器区间指定的其他容器中的多个元素

删除元素

  • 从容器末尾删除一个元素借助 pop_back()

pop_back() 方法删除元素但是不返还该元素，若是想得到尾部的元素需要先调用 back()

  • 清空容器所有元素借助 clear()

clear() 方法删除容器中所有的元素

  • 删除指定元素借助 erase()
    • 版本一：删除指定位置的元素
    • 版本二：删除迭代器器区间指定的多个元素

示例

容量和大小

内容分配机制

  • vector 容器具有“动态扩容”的功能，其实现的方法是先分配一定容量的内存，当插入的元素无法容纳的时候就会重新分配内存空间并将数据拷贝到新申请的内存中（数据要求在内存中连续）
  • 为了提高效率（不用每次都需要重新分配内存），重新分配内存是将一次性扩充较大的容量（一般是 2 倍）添加若干个元素后再重新分配

容量和大小

大小：size() 返还容器内实际存储的元素个数

容量：capacity() 再次分配内存前最多可以存放的元素数量

empty()：返还容器是否为空

一定不能混淆容量和大小：这是两个概念，容量指的是当前能容纳的最大体积，而大小指的是已经占用的体积

预留容量

  • 有时候为了减少重新分配内存带来的开销，可以在定义容器后预留足够的空间
  • reverse()：预留指定大小的空间
  • resize()：改变元素的数量，若是多出数量将调用缺省构造函数来填充

容器的容量扩展到 1000，但是实际元素的个数为0，相当于体检预留了指定的空间

容器内元素变为 1000 个，所有元素都通过缺省构造函数生成将容器的元素补充到 1000 个

双端队列 deque

deque 与 vector 类似但是使用相对较少，双端队列即可以在两边插入删除元素

  • 实现时元素在内存中不要求连续（也就不能通过下标运算符访问了）

  • 使用 deque 在头部和尾部插入和移除元素均能达到常量级别时间，而 vector 只能在尾部插入和移动元素时才能达到分摊的常量时间

  • deque 支持 push_front() pop_back() 和 pop_front() pop_back()

  • deque 不支持 reverse() 和 capacity()，即不能查看和修改容量

双向链表 list

特点

STL中的 list 是一种标准的双向链表

  • 支持常量时间的任意位置插入和删除操作
  • 对单个元素的访问为线性时间（访问速度慢）
  • 不支持下标运算符的访问，只能通过迭代器访问
  • 构造、析构、比较、赋值等运算与 vector 类似

元素访问

  • 头、尾元素的访问
    • front()：返还第一个元素
    • back()：返还最后一个元素
  • 其他元素的访问
    • 通过迭代器实现
    • 不支持随机访问，如下标运算符

添加删除元素

与 deque 类似

  • push_back()
  • pop_back()
  • push_front()：deque 支持但是 vector 不支持
  • pop_front()：deque 支持但是 vector 不支持
  • 3 个版本的 insert()
  • 2 个版本的 erase()
  • clear()

大小

与 deque 类似，list 也不暴露底层的内存模型

  • size()：返还元素的个数
  • empty()：判断容器是否为空
  • 不支持 reverse() 和 capacity()

拼接 splice

  • 通过 splice() 可以将一个 list 中的部分元素剪切出来并插入到另一个 list
  • 该操作可以在常量时间内完成
    • 版本一：剪切并插入一个元素
    • 版本二：剪切并插入指定范围内的所有元素

splice() 是一种破坏性的方法，源 list 中的元素将被剪切并移动到目标的 list 中

其他方法

  • remove() 和 remove_if()：将指定元素动 list 中删除

  • unique()：移除连续的重复元素

  • merge()：前提是两个 list 有序下合并两个 list，破坏源 list

  • sort()：排序 list

  • reverse()：翻转（逆序）

数组 array

C++ 11 引入的新容器，与 vector 类似大小固定。感觉就像是一个能够使用部分 STL 函数的数组而已

  • 构造后大小固定不能再增加或者收缩
  • 能够分配在栈上，vector 需要分配在堆上
  • 支持下标运算符 []、front()、back() 等方法访问
  • 不支持 push_back()、insert()、clear() 等方法

构造时需要指定数据类型和大小作为参数

数据类型相同大小不同的容器类型不同

单向链表 forward_list

C++ 11 引入的单向链表，与之前的 list 相比只能在尾部插入元素

  • 支持从头到尾的前向遍历

如果想要定位到最后一个元素只能从头遍历

forward_list::insert_after() 是 C++ STL 中的内置函数，它使我们可以选择在正向列表中给定迭代器指向的元素之后的位置插入元素

适配器容器

分类

queue priority_queue 和 stack

  • 适配器通过包装一种顺序容器实现，将顺序容器作为底层存储机制
  • 适配器容器结合应用的特点，对包装的顺序容器接口进行了简化和特化

单向队列 queue

特点

  • 队列提供“先进先出”的标准队列
  • 类型定义：template <typename Container = deque<T> > class queue;

存储的数据类型是 T，缺省的底层容器是 deque<T>

操作

方法 作用
push() 从队尾添加元素
pop() 从队首删除元素并且不返还
front() 获取队首元素
back() 获取队尾元素
size() 返回队列的元素数量
empty() 判断队列内是否为空

优先队列 priority_queue

特点

  • 优先队列保证队列中的元素有序

  • 按照规定将拥有最高优先级的元素位于队首，优先级相同的元素按照 FIFO 顺序出队

  • 类型定义：template <typename T,typename Container = vector <T>,typename Compare = less<T> >;class priority_queue

存储的数据类型是 T，缺省的底层容器是 vectore<T>，Compare 为排序时的比较运算**，**按照缺省值最大的元素优先出队

操作

方法 作用
push() 向队列中插入元素，插入后会引起排序
pop() 从队首删除元素并且不返还
top() 返回队首元素的常引用
size() 返回队列的元素数量
empty() 判断队列内是否为空

元素没有可以用于直接修改的函数或者间接获取元素本身的方法，所以插入时排序就可以保证队内自身有序

priority_queue 不支持 push_back() 方法，无法操作队尾，不支持迭代器，无法合并2个优先级队列

示例：错误处理

  • 需求：按照错误级别的顺序处理错误
  • 实现：用优先队列实现错误类，队首到队尾按照按照错误等级从小到大顺序排列

堆栈 stack

特点

  • 堆栈遵循 LIFO 后进先出的原则，实现标准的堆栈
  • 类型定义：template <typename T,typename Container = deque<T> > class stack;

存储的数据类型是 T，缺省的底层容器是 deque<T>

操作

方法 作用
push() 向队列中插入元素，插入后会引起排序
pop() 从队首删除元素并且不返还
top() 返回队首元素的常引用
size() 返回队列的元素数量
empty() 判断队列内是否为空

示例：错误处理

  • 基本需求
    • 改变按照错误级别的顺序处理错误
    • 调整为优先处理最近发生的错误
  • 实现：将原来实现的优先队列替换为堆栈，即可以实现按照最近发生的错误优先处理

关联容器

分类

map multimap set 和 multiset

  • 元素非线性存储
  • 提供键到值得映射
  • 将元素按照有序的、树状结构存储

基础类 pair

特点

  • pair 是 C++ 的一个基础模板类，它将两种不同类型的数据包装绑定在一起
  • 通过 first 和 second 访问其中的数据成员
  • 重载了 == 和 < 运算符
  • 包含在头文件 <utility> 中

示例

构建 pair

STL 中提供了 make_pair() 模板函数便于我们通过两个变量构建一个 pair 对象

或者就是向上述那样直接定义 pair

双列集合 map

特点

  • 存储“键/值” （pair）的关联容器，而不是存储单个数据
  • 提供插入、删除、查找功能，都是基于 key 的，而 value 只是绑定于特定键的附带值
  • 存储的 pair 按照 key 排序、插入、删除、查找的时间复杂度为对数复杂度
  • 底层存储中往往采用平衡树来存储但是对于用户透明

构建 map

key 的类型和 value 的类型，比较类型时：提供元素排序时的比较运算规则（键值 key 的大小），可以使用缺省的运算，要求 key 的类型支持 < 运算符

map 的初始化

C++ 11 支持在 map 声明的时候同时进行初始化

插入元素

特点

  • vector 和 list 等容器插入元素需要指定要在哪个位置之前插入，但是所有关联容器不关心插入的位置，只关心 key 和 value，由容器内部确定其插入的位置
  • map 提供了某个版本的 insert() 方法，可以指定插入的确切位置，而不是插入在该位置之前
  • map 要求每个元素 （pair）拥有独一无二的 key，如果希望多个元素对应相同的键，应该使用 multimap

insert()

  • 输入参数：由键和值构成的一个完整的 pair
  • 返还值
    • 返还值也是一个 pair
    • 其中 first（key）值为插入成功后对应的那个元素的迭代器
    • second（value）值为 bool 类型，表示成功插入与否，因为 insert() 不会改写已经存在在容器里的键值对

示例

  • 声明的 ret 为了接受 insert() 返还的结果。ret.first 对应插入成功的新元素的迭代器，ret.second 标识是否成功插入。

  • auto ret = scoreMap.insert({'A',90}); 简化了程序

  • 当键值已经存在的时候，insert() 因为不能修改已经存在的键值对，将 ret.second 置为 false

  • 插入失败的时候就不能再用了，ret.first 也就失去了意义

[] 运算符

特点

  • 重载 [] 运算符，以 key 为索引并直接对 value 进行修改
  • [] 运算符永远执行成功，若指定的 key 不存在则会插入新的 pair，若是指定的 key 已经存在，在修改其对应的 value

示例

使用 []，若是已经存在则会替换其 value

multimap

特点

  • 支持多个值对应同一个键
  • 和 map 接口类似但是不支持 []
  • 调用 insert() 向 multimap 总会成功返回值为新插入元素的迭代器
  • 查找元素的问题不支持 []，通过 find() 返还任意一个键值匹配的 pair

multimap 区间查找元素

博客转载

如果不想查找的元素返还一个随机的，我们可以借助 lower_bound() 和 upper_bound() 找到区间内指定的元素

multimap 将键值相同的元素存储在一起，通过遍历得到子区间进一步访问键相同的元素

  • lower_bound()：返还指向键值不小于 k 的第一个元素
  • upper_bound()：返还指向键值大于 k 的第一个元素
  • equal_range()：返还一个 pair，first 成员等价于 lower_bound() 而 second 成员等价于 upper_bound()

当然，这些操作返回的也可能是容器自身的超出末端迭代器，如果所查找的元素拥有 multimap 容器中最大的键，那么在该键上调用 upper_bound 将返回超出末端迭代器。如果所查找的键不存在，而且比 multimap 容器中所有的键都大，则 lower_bound 也将返回超出末端迭代器

map 也支持这些方法，但用处不大。就和 count() 一样而对于 map 和 set 并没有什么用处

set

特点

  • set 和 map 类似但是不存储键值对而只是存储本身，键和值合二为一，键本身就是要存储的信息
  • set 的接口与 map 类似但是不提供 []
  • 避免通过迭代器修改键值这会导致排序状态遭到破坏

multiset

特点

  • multiset 和 set 关系就如同 multimap 和 map
  • multiset 允许存储相同的元素
  • upper_bound()、lower_bound()等方法仍然有效

无序关联容器

C++ 11 对关联容器进行的改进得到了无序关联容器，他们不是使用比较运算符来组织元素，而是使用一个哈希函数和关键字类型的 == 运算符。在关键字类型的元素没有明显的序关系的情况下，无序容器也是非常有用的，在某些应用中维护元素的代价非常高昂，此时无序容器是一个有效的方法

分类

  • unordered_map、unordered_multimap、unordered_set 和 unordered_multiset

  • 对应关联容器的无序版本，插入、删除和查找更快

  • 继续哈希函数的散列表

  • 哈希函数将键转换为桶索引

unordered_map

特点

  • 构造是 5 个模板参数：键类型、值类型、散列类型、判等比较类型、分配器，后三个可以取缺省值

常见操作

bucket_count()：返还桶的数量

bucket(key)：返还键值对应桶的序号

local_iterator：可遍历指定桶中的元素

begin(n)：获得序号 n 桶的起始位置

end(n)：获得序号 n 桶的结束位置

unordered_multimap

特点

  • 与 unordered_map 接口类似但是不提供 []
  • 允许键值重复，插入永远成功
  • 通过 equal_range() 方法返还指定值的迭代器区间

unordered_set

  • unordered_set 和unordered_multiset 的关系类似于 set 和 multiset

  • 不会对键值进行排序

其他容器

普通数组作为容器

普通数组具有 STL 容器的部分属性

  • 通过指针遍历和访问数组元素，但是数组不支持 size()、insert() 等操作，不是真正意义上的容器
  • 可以将指针看做是迭代器所以能够用于部分算法和容器的操作

将普通数组的指针看做是迭代器可以将数组内的元素插入到 vector 中

string 作为容器

STL 中提供 string 类专门用于处理字符串，string 也支持部分容器的接口：begin()、end()、insert()、erase()、size()、empty()、reverse()、capacity()、push_back() 等
