二分与三分查找
发表于2020-07-29|更新于2020-11-27|ACM学习基础算法
|字数总计:1.7k|阅读时长:6分钟|阅读量:

查找：分为整数查找和浮点数查找，方法是通过控制循环次数或者精度来终止

适用前提：线性表，有序表

原理：查找的原理是利用区间内值有序的特点, 不断让可行区间减半, 最终可行区间长度减到 1 得到答案

二分查找

整数二分查找

最难的就是整数二分查找，坑点主要在于如果没有控制好查找终止边界有很大的可能出现死循环

保证终止的条件：

  • 解一直在可行区间里
  • 每次判断后可行区间都会缩小（特别是左右端点相距为1的时候）

第一点很好保证, 第二点在可行区间长度大于 2 的时候不会出问题。但是当可行区间左右端点相距为 1 时未能处理好中间值取法（由于是整数所以中间值 mid 取整后可能还是左右端点导致可行区间没有缩小）可能陷入死循环

因此重点要考虑的是写好的二分查找左右端点为 1 的时候找到中间值后可以继续缩小区间

我们写二分的时候, 可以先保证第一点, 让解一直在可行区间里面, 然后再根据左右端点相距 1 的时候的情况,特殊调整 mid 到底是向上还是向上取整, 是 +1 还是 -1

综上影响因素：

  • 区间的开闭
  • 左右端点开闭情况
  • 中间点是取整还是进一
  • 端点能否取得
  • 重复的数取第几个

模板

用于在 [l,r) 内查找是否存在 key 的值并返还相应位置，没有返还 -1

虽然这个模板用于左闭右开的，但是其他的区间查找只要套合理就可以了。例如若是开区间 (l,r) 说明左端点无法取得，那么直接套用 bsearch(l+1,r) 就好了

方法一：将 [l,r) 划分为区间 [l,mid] 和 [mid+1,r) ，更新操作为 r=mid 或者 l=mid+1 ，此时 mid=(l+r)>>1

当 [l,r) 相距为1时，若是 a[mid]>key 下次取 mid 就会成功取到左端，若是 a[mid]<key 下次区间相距为 0 也可以成取到

首推这个写法不容易错

方法二：将区间 [l,r) 划分为区间 [l,mid-1] 和 [mid,r) ，更新操作为 r=mid-1 或者 l=mid ，此时 mid=(l+r+1)>>1

注意：mid 的取法必须是 l+r+1>>1 因为若是不多加那个 1 会在区间相距为 1 的时候陷入死循环

当 [l,r) 相距为1时，若是 a[mid]<key 下次取 mid 恰好因为多加个1成功取到右端，若是 a[mid]>key 下次区间距离为 0 也可以成功取到

找上下界问题

不同于找 key 的位置问题，这类问题是找到关于 key 的上界和下界，说明肯定是存在能找得到没有查找失败的可能

下面给的模板以上述方法一为基础，划分区间为 [l,mid] 和 [mid+1,r)

二分法求上界

在表中找到第一个大于 key 的位置（不存在就返还表的最后一个数的后一个位置），作用相当于 upper_bound()

二分法求下界

在表中找到第一个大于等于 key 的位置（不存在就返还表的最后一个数的后一个位置），作用相当于 lower_bound()

浮点数二分查找

相比于整数的二分,浮点数的二分好写的多，因为是找区间内一个不确定的大概的数肯定存在，所以它没有取整的问题也不涉及边界是否可取问题（因为肯定是在中间取得）

模板

方法一：以循环次数为循环终止条件

通过控制循环的次数来取得要求精度，60 次循环达到 1e-18 。100 次的循环则可以达到 1e-30 的精度范围，基本上足以解决所有问题

因为有循环限制所以只会出现精度不够，不会出现死循环首推这个写法简单安全

方法二：先定义好终止循环的精度范围

通过限制左右区间相差的距离从而限制 mid 精度

三分查找

三分查找由浮点数二分查找衍生过来，主要是解决查找凸函数的最值问题

  • 如果 lm 和 rm 在最大（小）值的同一侧： 那么由于单调性，一定是二者中较大（小）的那个离最值近一些，较远的那个点对应的区间不可能包含最值，所以可以舍弃。
  • 如果在两侧： 由于最值在二者中间，我们舍弃两侧的一个区间后，也不会影响最值，所以可以舍弃。
